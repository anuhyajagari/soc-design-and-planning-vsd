## 1. Where is a Chip Located & Understanding its Components

### Chip Location
Before diving into the internal architecture, it is essential to understand where the chip resides. A finalized integrated circuit (IC) or chip package is surface-mounted directly onto a **PCB (Printed Circuit Board)**. The PCB contains copper tracks that route electrical power and communication signals between the chip pins and other peripheral components on the system board.

### Inside the Chip
Peeling back the outer packaging of the chip reveals the following structures:

* **Die:** The actual piece of silicon that contains the complete circuit implementation.
* **Core:** The central area of the chip where standard cells, logic gates, and digital circuitry are placed.
* **Pads:** Peripheral connection blocks located around the boundary of the die that provide connections between the internal circuitry and external pins.
* **Macros:** Large, complex, pre-designed functional blocks (such as SRAMs, PLLs, and processors) handled manually during placement.
* **IPs (Intellectual Property Blocks):** Reusable and pre-verified design blocks integrated into a chip design.
* **Foundry:** The semiconductor manufacturing company responsible for fabricating the chip.

## 2. RISC-V ISA: From Software App to Physical Layout

How a piece of software code actually communicates with and runs on physical hardware:

* **C Program:** The high-level code written by a human (easy for us to read, impossible for a chip to run).
* **Compiler:** Breaks down the high-level C code into **Assembly Code** instructions specific to the processor.
* **Assembler:** Converts that text assembly code directly into binary **RISC-V Machine Instructions** (`0`s and `1`s) that the CPU hardware recognizes.
* **RTL Implementation:** The physical architecture code (written in Verilog) that defines the actual internal circuits, registers, and logic needed to execute those RISC-V instructions.
* **Netlist:** The structural gate-level blueprint generated by compiling the RTL, showing exactly which logic gates connect together.
* **Physical Layout:** The final coordinates mapping out where the actual metal layers, transistors, and silicon are printed onto the die.

## 3. Components Required for Open-Source ASIC Design

To successfully execute an automated open-source digital design flow, three fundamental pillars must be present:

### 1. RTL Design
The hardware description code (Verilog) defining the logic and behavior of the target design.

### 2. EDA Tools
Open-source automated software suites used for synthesis, placement, routing, and physical sign-off verification. 
* *Examples:* OpenLANE, OpenROAD, Yosys, Magic, Netgen.

### 3. PDK (Process Design Kit)
The collection of files that links software tools to a real manufacturing foundry. It contains:
* Design rules (DRC, LVS)
* Device simulation models
* Standard Cell Libraries (SCL)
* Timing and physical layer information
* *Workshop PDK:* **Sky130 Open-Source PDK**

## 4. RTL to GDSII Flow & Detailed Physical Design Stages

The ultimate objective of physical design is converting an abstract text design file down into a verified manufacturing blueprint layout database (**GDSII** format).

### Core Pipeline Execution Stages:

* **RTL (Register Transfer Level):** The baseline Verilog code describing how the chip should behave.
* **Synthesis:** Translates the Verilog code into a **Gate-Level Netlist** using real standard cells from the library.
* **Floorplanning:** Sets the physical chip size boundaries, places I/O pins, and fixes large macro blocks in place.
* **Power Planning:** Builds the Power Distribution Network (PDN) using power rings and straps to deliver power evenly and reduce **IR Drop**.
* **Placement:** Automatically arranges the tiny standard cells into organized horizontal layout rows while minimizing total wire length.
* **Clock Tree Synthesis (CTS):** Builds a balanced clock distribution network to distribute the timing signal evenly and minimize **clock skew**.
* **Routing:** Lays down the final physical metal wires layer-by-layer to interconnect all standard cells together along a grid.
* **Sign-Off Checks:** * *STA (Static Timing Analysis):* Verifies the circuit meets all speed and setup/hold timing limits.
  * *DRC (Design Rule Check):* Ensures no geometric lines violate foundry rules (prevents short and open circuits).
  * *LVS (Layout vs Schematic):* Verifies that the physical layout wiring matches the logical netlist connections 100%.
* **GDSII:** The final layout blueprint database streamed out and sent to the foundry to manufacture the physical chip.

### Lab — Running OpenLANE for picorv32a

#### Setting Up and Invoking OpenLANE

```
cd /home/vscode/Desktop/OpenLane
make mount
./flow.tcl -interactive
package require openlane 1.0.2
```


#### Preparing the Design

```
prep -design picorv32a
```



#### Running Synthesis

```
run_synthesis
```



#### Flop Ratio Calculation

After synthesis completes, the flop ratio can be calculated from the synthesis statistics:

```
Flop Ratio = Number of D Flip Flops / Total Number of Cells
           = 1613 / 15762
           = 0.1023
           = 10.23%
```
